# Аутентификация и доступ

<!-- FASTNEAR_AI_DISCOVERY: Агентам и automation-сервисам стоит использовать /agents/auth и не хранить учётные данные в браузерном хранилище. -->

API-ключи FastNear решают две разные задачи:

1. Быстрые демо внутри документации, где нужно запускать встроенные примеры в пределах лимитов вашего аккаунта.
2. Доступ из продакшен-бэкенда, где ваш сервис, агентный раннер или прокси должен полностью контролировать жизненный цикл учётных данных.

Рассматривайте их как отдельные сценарии.

## Выберите подходящий сценарий

- [Браузерное демо](/auth/browser-demo): используйте встроенный браузерный поток документации, чтобы быстро пробовать запросы. Это режим для демо, локальной разработки и личных экспериментов.
- [Продакшен-бэкенд](/auth/backend): храните ключи на серверной стороне, подставляйте их из переменных окружения или хранилищ секретов и никогда не раскрывайте в браузере.

## Что представляет собой эта страница и чем она не является

- Эта страница — самый быстрый способ опробовать аутентифицированные примеры из документации и понять доступные схемы аутентификации.
- Эта страница не рекомендует хранить продакшен-учётные данные в браузерном `localStorage`.
- Если ваш код работает в браузере, расширении или публичном клиентском приложении, считайте такой секрет раскрытым.

## Получение API-ключа

1. Откройте [подписки FastNear](https://dashboard.fastnear.com/#pricing).
2. Выберите тариф, который подходит под вашу нагрузку.
3. Сгенерируйте API-ключ в FastNear Dashboard.
4. Используйте браузерный демо-сценарий только для локального знакомства с документацией.
5. До выхода в продакшен переведите рабочий трафик на серверную схему.

## Когда вам нужно больше, чем демо

- Используйте [Продакшен-бэкенд](/auth/backend) для переменных окружения, прокси-паттернов, редактирования секретов и ротации ключей.
- Используйте [Аутентификацию для агентов](/agents/auth), когда доступ FastNear нужен агентному раннеру или сервису автоматизации.
