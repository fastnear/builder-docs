# Справочник API

Этот раздел объясняет, что относится к REST-слою FastNear и как выбрать между REST-семействами до перехода к сырому JSON-RPC.

Сайдбар `/api` намеренно посвящён именно **FastNear API**, а не универсальному REST-разделу. Другие REST-семейства — [Транзакции API](/tx), [API переводов](/transfers), [NEAR Data API](/neardata) и [KV FastData API](/fastdata/kv) — живут в собственных верхнеуровневых разделах.

## REST-семейства с одного взгляда

| Семейство | Начните здесь, когда... | Переключайтесь, когда... |
| --- | --- | --- |
| [FastNear API](/api) | нужны индексированные представления аккаунтов, токенов, NFT, стейкинга или публичных ключей | требуется каноническая семантика RPC на уровне протокола |
| [Транзакции API](/tx) | нужна история транзакций, квитанций, аккаунта или блока | нужны только события переводов или точное поведение на уровне RPC |
| [API переводов](/transfers) | вопрос именно о движении NEAR или FT-токенов | вопрос расширяется до общей истории исполнения |
| [NEAR Data API](/neardata) | нужны свежие оптимистичные или финализированные чтения семейства блоков | нужно точное каноническое продолжение по блоку или состоянию |
| [KV FastData API](/fastdata/kv) | нужна индексированная история «ключ–значение» или последнее индексированное состояние ключа | нужно точное текущее состояние контракта в цепочке |

## Для чего нужен сам `/api`

Используйте раздел [FastNear API](/api), когда пользователю нужен ответ в продуктовой форме без сшивания сырых ответов узла:

- сводки по аккаунту
- балансы FT-токенов
- активы NFT
- позиции стейкинга
- разрешение публичного ключа в аккаунт

Хорошие стартовые страницы в этом сайдбаре:

- [V1 Full Account View](/api/v1/account-full)
- [V1 Account FT](/api/v1/account-ft)
- [V1 Account NFT](/api/v1/account-nft)
- [V1 Account Staking](/api/v1/account-staking)
- [V1 Public Key](/api/v1/public-key)

## Когда не стоит стартовать с `/api`

Не начинайте с этого сайдбара, когда:

- основная задача — история; используйте [Транзакции API](/tx) или [API переводов](/transfers)
- основная задача — опрос свежих блоков; используйте [NEAR Data API](/neardata)
- ответ должен оставаться близким к каноническим формам запроса и ответа узла; используйте [Справочник RPC](/rpc)
