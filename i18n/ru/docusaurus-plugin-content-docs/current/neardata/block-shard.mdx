# Шард блока

Используйте этот маршрут, когда данных одного чанка уже недостаточно и нужно более полное представление одного шарда для финализированного блока.

Этот маршрут идёт на шаг шире, чем [Чанк блока](/neardata/block-chunk): помимо самого содержимого чанка, вы ещё видите изменения состояния на шарде и квитанции, которые это исполнение произвело. Поэтому этот маршрут лучше подходит для отладки, индексаторов и аналитики, когда важны итоги исполнения, а не только включённые транзакции и входящие квитанции. За готовым сценарием, который начинается с транзакции и заканчивается конечным чанком, переходите в [Примеры RPC](/rpc/examples).
