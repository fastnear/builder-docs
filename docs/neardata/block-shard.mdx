# Block Shard

Use this when chunk data alone is not enough and you need the fuller shard envelope for a finalized block.

`Block Shard` goes one step wider than [Block Chunk](/neardata/block-chunk): you still get the shard's chunk payload, but you also see shard state changes and the receipts that execution produced. That makes it the better surface for debugging, indexers, and analytics when execution outcomes matter, not just included transactions and incoming receipts. For a worked tracing flow that starts from a transaction and ends on a destination chunk, see [RPC Examples](/rpc/examples).
