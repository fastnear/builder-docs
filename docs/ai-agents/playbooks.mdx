# Agent Playbooks

## Account research

- Start with [FastNear API](/docs/api) for balances, NFTs, staking, and public-key resolution.
- Drop to [RPC Reference](/docs/rpc) only when you need canonical state details the indexed view does not expose.

## Transaction investigation

- Start with [Transactions API](/docs/tx) for account history, block lookups, and receipt investigation.
- Fall back to [RPC Reference](/docs/rpc) for canonical status checks or transaction submission-related workflows.

## Near-realtime monitoring

- Poll [NEAR Data API](/docs/neardata) for optimistic or recent finalized block-family reads.
- Keep the polling logic explicit. This is not a websocket or webhook surface.

## Operator bootstrap

- Use [Snapshots](/docs/snapshots/) when the workflow is about standing up infrastructure, not when the agent simply needs chain data.
